# Cascading
CSS는 Cascading Style Sheet의 줄임말입니다.  
> Cascading은 폭포, 위에서 아래로 쏟아지는 이라는 뜻을 가진 단어입니다.
Cascading은 `스타일 우선순위`, `스타일 상속` 이 두가지 원칙을 가지고 있습니다.  

**스타일 우선순위**  
1. 중요도 : 스타일이 선언된 위치에 따라 우선순위가 결정됩니다.
2. 명시도 : 셀렉터(선태자)에 따라 우선순위가 달라지는 것입니다. `인라인>id>class>태그`
3. 코드 순서 : 나중에 온 속성이 최우선으로 적용됩니다.

**스타일 상속**  
부모요소의 스타일을 자식 요소들이 받아서 갖게 됩니다.

# display
HTML 요소를 어떻게 보이게 할지를 결정합니다.  
**display:none 과 visibility:hidden 의 차이점**
| 구분                | 차이점                                                                                |
| ------------------- | ------------------------------------------------------------------------------------- |
| display:none        | 보이지도 않고 해당 공간도 존재하지 않게 됨                                            |
| visibility : hidden | 보이지만 않고 해당 곤간은 존재, width, height 값을 주었다면 그만큼 공간은 존재하게 됨 |

⭐ 속성리스트
- block : 블록 속성
- inline : 인라인 속성
- inline-block : block과 inline의 중간 형태
- flex : 아이템들을 가로 혹은 세로 방향으로(1차원)배치할 수 있는 방식
- grid : 2차원으로 배치하는 방식으로 column과 row의 비율이나 크기를 지정합니다.

# margin과 padding에 대해 설명해주세요.

- margin은 바깥쪽 여백, padding은 안쪽 여백을 의미합니다.  
- 각각 border를 경계로 나뉩니다.
# Box model에 대해 설명해주세요.

> 모든 HTML요소는 Box 모양으로 구성되며, 이것을 박스 모델이라고 부릅니다.
박스 모델은 HTML요소를 padding, border, margin, content로 구분합니다.

- content : 텍스트나 이미지가 들어있는 박스의 실질적인 내용 부분
- padding : 내용과 border 사이의 간격
- border : 내용과 패딩 주변을 감싸는 테두리
- margin : 테두리와 이웃하는 요소 사이의 간격

# ⭐ position 속성
> 문서 상에 요소를 배치하는 방법을 지정합니다.

- `static` 
  - 기본값입니다. 요소를 문서 흐름에 맞게 배치합니다.
- `relative`
  - 요소를 기존 위치에서 상대적인 위치로 이동합니다.
  - 다른요소에 영향을 주지 않으며 정상적인 문서 흐름에 따라 요소가 배치됩니다.
- `absolute`
  - 요소를 가장 가까운 상위 요소 중 `position : relative || absolute || fixed` 중 하나인 요소를 기준으로 이동시킵니다.
  - 정상적인 문서 흐름에서 제외된다.
- `fixed`
  - 요소를 브라우저 창을 기준으로 이동시킵니다.

# Flexbox , Grid 특징

- FlexBox
  - 단일 차원(행 또는 열)에서 요소를 배치할 수 있습니다.
- Grid
  - 2차원 레이아웃을 처리할 수 있으며, 행과 열을 사용하여 요소를 배치할 수 있습니다.
  - 브라우저 창을 줄여도 자동으로 리스트의 width가 퍼센트(%)를 준것 처럼 브라우저 창에 반응합니다.

# float 동작에 대해 설명해주세요

> 뜨다 라는 의미이며 원래 웹페이지에서 이미지를 어떻게 띄워서 텍스트와 함께 배치할 것인가에 대한 속성입니다.
  
float속성을 사용하면 요소가 왼쪽이나 오른쪽으로 이동하면서 흐름에서 빠져 다른 요소들이 그 주위를 감싸게 됩니다.

# 반응형웹과 적응형웹에 대하여
- 반응형 웹(Responsive Web Design)
  - 하나의 웹사이트에서 PC, 스마트폰, 태블릿 PC 등 접속하는 디스플레이의 종류에 따라 화면의 크기가 자동으로 변하도록 만든 웹페이지 접근 기법
- 적응형 웹(Adaptive Web Design)
  - 반응형 웹의 반대라고 할 수 있다.
  - 서버 측 스크립트를 사용하여 디바이스의 속성(해상도, 화면 크기 등)을 감지하여 적절한 레이아웃과 디자인을 선택합니다.

# 반응형 웹의 3요소
1. 그리드 레이아웃(유동 그리드 : Fluid Grids)
2. 가변형 이미지(유동 이미지 : Fluid Images) 
3. 미디어 쿼리(Media Queries)

# px, em, rem
- px : 스크린의 물리적인 픽셀 단위입니다. 고정된 크기를 가지며 화면 해상도나 디스플레이 크기에 따라 크기가 변하지 않습니다.
- em : 현재 요소의 `font-size`값에 따라 크기가 상대적으로 결정됩니다.
- rem : 최상위 요소인 `html`요소의 `font-size`값에 따라 상대적으로 결정됩니다.
  - `em`과 다르게 부모 요소의 크기가 아닌 최상위 요소의 크기에 따라 결정되므로, 크기를 일관성 있게 유지하기 쉽습니다.

# CSS 적용 우선순위
CSS 스타일 규칙이 충돌하는 경우, 브라우저는 스타일 규칙의 우선 순위에 따라 적용합니다.  
1. important
2. 명시도(Specificity): 더 구체적인 선택자가 더 높은 우선 순위를 가집니다.
   - 선택자의 명시도는 ID 선택자(1점), class 선택자(0,1점), 태그 선택자(0.01점)에 따라 계산됩니다.
   - 예를 들어 `#my-div`는 `.my-class`보다 높은 우선 순위를 가집니다.
3. 선언 순서(Source order) : 동일한 우선 순위를 가진 규칙은 나중에 선언된 규칙이 덮어씁니다.

# CSS-in-JS
> 자바스크립트 코드에서 CSS를 작성하는 방식을 말합니다.

- 장점
- 단점



```jsx
import styled from 'styled-components';

const Text = styled.div`
  color: white,
  background: black
`

<Text>Hello CSS-in-JS</Text>
```
브라우저에서 DOM 노드를 다음과 같이 연결합니다:
```html
<style>
.hash136s21 {
  background-color: black;
  color: white;
}
</style>

<p class="hash136s21">Hello CSS-in-JS</p>
```

# CSS 전처리기

> 전처리기는 원래의 코드를 수정하거나 보완하는 데 사용되는 프로그램입니다.

```
sass / less / ...
```
- 장점 : 조건문이나 반복문, 간단한 연산 등을 할 수 있어서 CSS를 마치 프로그래밍 하듯 코딩할 수 있다.
- 단점 : 웹에서는 CSS만 동작하기 때문에 전처리기는 직접 동작시킬 수가 없다. 따라서 CSS 로 컴파일 후 동작시켜야 한다.

# 가상 클래스에 대해 설명해주세요.

> 가상 클래스(pesudo-class)는 선택자에 추가되어 해당 요소가 특정 상태에 있을 때 스타일을 적용하는 방법입니다.

| 가상 클래스   | 내용                                   |
| ------------- | -------------------------------------- |
| :hover        | 마우스 커서가 요소 위로 올라가 있을 떄 |
| :click        | 요소가 클릭될 떄                       |
| :first-child  | 부모 요소의 첫 번쨰 자식 요소일 때     |
| :nth-child(n) | 부모 요소의 n번쨰 자식 요소일 때       |

# margin 병합에 대해 설명해주세요.

> 인접하는 블록 요소 상하단의 마진이 병합되는 현상을 말합니다.
> 마진의 크기는 병합되는 마진 중에서 큰 값을 가진 마진의 값으로 병합됩니다.

**조건**
- 인접하는 Block 요소끼리만 일어납니다
- 상하단만 병합현상이 일어납니다.(좌우의 여백 병합은 일어나지 않음)

해결법은 border나 padding속성을 주는 것으로도 병합현상을 없앨 수 있습니다.
